# Contributing to FlowNeuro

Thank you for your interest in contributing to Flow.

## How to Contribute

### 1. Reporting Bugs
Open an issue in the respective repository if you encounter a bug.
* Verify the issue has not already been reported.
* Include your device model, OS version, and application version.
* Provide steps to reproduce the issue. Include logs or screenshots if applicable.

### 2. Suggesting Features
When requesting a feature:
* State the problem the feature addresses.
* Ensure the request aligns with the project's local-first architecture. Features requiring external cloud tracking, proprietary SDKs, or remote servers will be rejected.

### 3. Submitting Pull Requests
To contribute code:
1. Fork the repository and create a branch from `main`.
2. Ensure your code follows the existing architecture (e.g., separating UI logic from the core engine).
3. Test your changes locally.
4. Open a Pull Request detailing the changes made and the reasoning behind them.

### 4. Translations
Translations are managed via Weblate. You can contribute to localization [here](https://hosted.weblate.org/engage/flow/)

## Development Environment Setup
Refer to the `README.md` in the specific repository (Android or Desktop) for build instructions, SDK requirements, and architecture documentation.
